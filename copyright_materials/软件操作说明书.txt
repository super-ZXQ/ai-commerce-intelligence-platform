1. 软件概述

1.1 软件简介

电商数据分析系统是一款基于 Python 开发的电商数据全链路分析平台，采用前后端分离架构设计。系统基于 102,318 条真实电商订单数据，经过数据清洗后保留 100,286 条有效记录，涵盖 78,060 名独立用户在 6 个平台（APP、微信公众号、Web 网站、淘宝、微信小商店、wap 网站）上的全年消费行为数据，总销售额达 101,776,848.74 元。

系统提供从数据清洗、BI 可视化、AI 智能查询到 RESTful API 服务的完整解决方案，旨在帮助电商企业实现数据驱动的精细化运营决策。

1.2 核心功能模块

本系统包含四大核心功能模块：

（1）BI 数据看板
基于 Streamlit 和 Plotly 构建的交互式数据大屏，支持多维度交叉筛选和实时数据可视化。提供核心指标卡片、销售趋势图、平台占比分析、RFM 用户分层等功能，帮助管理者快速掌握业务全貌。

（2）AI 分析助手
基于 LangChain SQL Agent 和通义千问大语言模型的自然语言查询系统。用户无需编写 SQL，直接用中文提问即可获取数据查询结果和可视化图表，大幅降低数据分析门槛。

（3）FastAPI 后端接口
提供 22 个 RESTful API 接口，涵盖认证、订单查询、商品分析、数据导出、AI 查询和系统监控等功能。采用 JWT 认证、API 限流、响应缓存等安全机制，支持外部系统（小程序、移动端、前端）实时调用数据。

（4）数据分析 Notebook
基于 Jupyter Notebook 的数据清洗与多维分析模块，包含数据加载与清洗、销售维度分析、时间维度分析、用户维度分析和综合分析 5 个独立 Notebook，完整记录数据分析过程。

1.3 技术架构

系统采用分层架构设计：

表现层：Streamlit BI 看板 + HTML 可视化页面 + Swagger API 文档
接口层：FastAPI 异步 RESTful API（22 个接口）
业务层：订单服务 + 分析服务 + AI 服务
数据层：SQLAlchemy Async ORM + aiomysql 连接池 + MySQL 8.0
安全层：JWT 认证 + API 限流 + SQL 注入防护 + 敏感数据过滤
基础设施：LangChain SQL Agent + 通义千问 LLM + 内存缓存

1.4 关键数据指标

原始数据量：102,318 条电商订单记录
清洗后数据：100,286 条有效记录（删除 2,032 条异常数据）
分析用户数：78,060 名去重独立用户
时间跨度：2025 年 1 月至 2026 年 1 月（全年销售数据）
总销售额：101,776,848.74 元（实际付款金额）
复购率：25.39%（消费 2 次及以上的用户占比）
支持平台：6 个（APP/微信公众号/Web 网站/淘宝/微信小商店/wap 网站）
API 接口：22 个（含认证/限流/监控）
测试用例：27 个异步单元测试

3. 安装与部署

3.1 安装前准备

在安装本系统之前，请确保已完成以下准备工作：

步骤一：安装 Python 3.11+
  访问 https://www.python.org/downloads/ 下载并安装 Python 3.11 或更高版本。
  安装时勾选"Add Python to PATH"选项。
  安装完成后，打开命令行执行以下命令验证：
    python --version
  应显示 Python 3.11.x 或更高版本号。

步骤二：安装 MySQL 8.0+
  访问 https://dev.mysql.com/downloads/ 下载并安装 MySQL 8.0。
  安装时设置 root 用户密码并记录。
  安装完成后，执行以下命令验证：
    mysql --version
  应显示 mysql Ver 8.0.x 或更高版本号。

步骤三：安装 Git
  访问 https://git-scm.com/downloads 下载并安装 Git。
  安装完成后执行：
    git --version

3.2 项目安装

步骤一：获取项目代码
  打开命令行终端，执行以下命令克隆项目代码：
    git clone https://github.com/2681107509-dev/ecommerce-analysis-system.git
    cd ecommerce_analysis

步骤二：创建 Python 虚拟环境
  执行以下命令创建并激活虚拟环境：
  Windows 系统：
    python -m venv .venv
    .venv\Scripts\activate
  Linux/macOS 系统：
    python3 -m venv .venv
    source .venv/bin/activate
  激活成功后，命令行提示符前会显示 (.venv) 标识。

步骤三：安装项目依赖
  依次执行以下命令安装各模块依赖：
    pip install -r requirements.txt
    pip install -r ai-ecommerce-assistant/requirements.txt
    pip install -r backend/requirements.txt
  安装过程中如遇网络问题，可使用国内镜像源：
    pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

3.3 数据库配置

步骤一：创建数据库
  登录 MySQL 命令行：
    mysql -u root -p
  执行以下 SQL 创建数据库：
    CREATE DATABASE ecommerce_analysis DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    USE ecommerce_analysis;

步骤二：创建数据表
  执行建表脚本：
    source sql/create_table.sql;
  该脚本将创建 orders 表，包含以下字段：
    id（主键）、order_no（订单编号）、user_name（用户名）、product_id（商品编号）、
    product_name（商品名称）、quantity（数量）、unit_price（单价）、
    total_amount（总金额）、payment_amount（实付金额）、order_time（下单时间）、
    order_date（订单日期）、order_status（订单状态）、payment_method（支付方式）、
    platform_type（平台类型）、is_refunded（是否退款）、refund_amount（退款金额）

步骤三：导入数据
  执行数据导入脚本：
    source sql/import_data.sql;
  导入完成后验证：
    SELECT COUNT(*) FROM orders;
  应返回约 100286 条记录。

步骤四：创建数据库索引
  执行索引优化脚本：
    source backend/sql/optimize_indexes.sql;
  该脚本将创建 7 个关键索引，加速常用查询操作。

3.4 环境变量配置

步骤一：创建配置文件
  复制示例配置文件：
    copy backend\.env.example backend\.env    (Windows)
    cp backend/.env.example backend/.env      (Linux/macOS)

步骤二：修改配置项
  使用文本编辑器打开 backend/.env 文件，修改以下配置：
    DB_HOST=localhost          # 数据库地址
    DB_PORT=3306               # 数据库端口
    DB_USER=root               # 数据库用户名
    DB_PASSWORD=你的密码        # 修改为实际 MySQL 密码
    DB_NAME=ecommerce_analysis # 数据库名称
    JWT_SECRET=your-secret-key # 修改为安全的密钥（生产环境必须修改）
    LLM_API_KEY=sk-xxx         # AI 助手 API 密钥（可选，不配置则 AI 功能不可用）
    LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
    LLM_MODEL=qwen-plus

3.5 启动服务

步骤一：启动 FastAPI 后端服务
  在项目根目录执行：
    python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
  参数说明：
    --host 0.0.0.0：允许所有 IP 访问
    --port 8000：指定服务端口
    --reload：开发模式，代码修改后自动重启
  启动成功后控制台显示：
    INFO:     Uvicorn running on http://0.0.0.0:8000

步骤二：启动 BI 数据看板（可选）
  新开终端窗口执行：
    streamlit run app/bi_dashboard.py
  浏览器自动打开 http://localhost:8501

步骤三：启动 AI 分析助手（可选）
  新开终端窗口执行：
    streamlit run ai-ecommerce-assistant/app.py
  浏览器自动打开 http://localhost:8501

4.3 AI 分析助手

4.3.1 功能概述

基于 LangChain SQL Agent 和通义千问大语言模型的自然语言查询系统。用户无需编写 SQL 语句，直接用中文提问即可获取数据查询结果和可视化图表。

4.3.2 核心能力

Text-to-SQL：
  用户输入自然语言问题，系统自动生成对应的 SQL 查询语句。
  示例：输入"最近7天的总销售额" -> 自动生成 SELECT SUM(payment_amount) FROM orders WHERE ...

智能图表生成：
  系统根据查询结果自动识别数据特征，选择最合适的图表类型：
  - 时间序列数据 -> 折线图
  - 分类对比数据 -> 柱状图
  - 占比分析数据 -> 饼图
  - 排名数据 -> 横向柱状图

敏感数据过滤：
  自动拦截涉及隐私的查询请求（密码、手机号、身份证号等），返回脱敏提示。

异常预警：
  自动检测数据中的异常指标（如异常高销售额、异常退款率等），并给出业务建议。

数据导出：
  支持将查询结果导出为 CSV 文件下载。

4.3.3 操作步骤

1. 启动 AI 助手服务：streamlit run ai-ecommerce-assistant/app.py
2. 浏览器自动打开 http://localhost:8501
3. 在页面底部的输入框中输入自然语言问题
4. 按 Enter 键或点击发送按钮
5. 系统显示执行步骤：理解问题 -> 生成 SQL -> 执行查询 -> 生成图表
6. 查看结果表格和可视化图表
7. 如需导出，点击"导出 CSV"按钮

4.7 认证系统

4.7.1 功能概述

基于 JWT Bearer Token 的认证系统，保护敏感接口（如 AI 查询）的访问安全。提供用户登录、Token 刷新和当前用户查询三个接口。

4.7.2 认证流程

完整认证流程如下：

步骤一：用户登录
  调用 POST /api/auth/login 接口
  请求体：{"username": "admin", "password": "admin123"}
  返回：{"access_token": "eyJhbGciOiJIUzI1NiI...", "token_type": "bearer", "expires_in": 86400}

步骤二：携带 Token 访问受保护接口
  在请求头中添加：Authorization: Bearer eyJhbGciOiJIUzI1NiI...

步骤三：Token 过期前刷新
  调用 POST /api/auth/refresh 接口（需携带当前 Token）
  返回新的 access_token

步骤四：查询当前用户信息
  调用 GET /api/auth/me 接口（需携带 Token）
  返回：{"username": "admin", "authenticated": true}

4.7.3 默认账号

系统预置两个默认账号：

  管理员账号：
    用户名：admin
    密码：admin123
    权限：全部接口

  分析师账号：
    用户名：analyst
    密码：analyst123
    权限：全部接口

4.7.4 安全说明

- 密码采用 SHA256 哈希存储，不以明文保存
- Token 采用 HS256 算法签名，有效期 24 小时
- 登录接口限流 5 次/分钟，防止暴力破解
- Token 过期后需重新登录获取

6. 安全机制说明

6.1 JWT 认证体系

系统采用 JWT（JSON Web Token）Bearer Token 认证机制，保护敏感接口的访问安全。

认证流程：
  1. 客户端使用用户名和密码调用登录接口
  2. 服务端验证凭据后签发 JWT Token（HS256 算法签名）
  3. 客户端在后续请求的 Authorization 头中携带 Token
  4. 服务端中间件验证 Token 有效性

密码安全：
  - 密码采用 SHA256 哈希算法存储
  - 不以明文形式保存或传输密码
  - 密码验证使用恒定时间比较，防止时序攻击

Token 管理：
  - Token 有效期 24 小时（86400 秒）
  - 支持 Token 刷新机制
  - Token 过期后需重新登录

6.2 API 请求频率限制

系统实现基于路径差异化的 API 限流机制，防止接口被恶意调用：

  登录接口：5 次/分钟（防止暴力破解）
  AI 查询接口：10 次/分钟（控制 LLM 调用成本）
  数据导出接口：3 次/2 分钟（防止大量数据导出）
  默认限制：100 次/分钟

超限响应：
  HTTP 状态码：429 Too Many Requests
  响应头：Retry-After: <秒数>
  响应体：{"detail": "请求过于频繁，请稍后再试"}

6.3 SQL 注入防护

系统在数据查询层实现 SQL 注入防护：

  - 所有查询使用 SQLAlchemy ORM 参数化查询
  - LIKE 查询中的 % 和 _ 特殊字符自动转义
  - 不使用字符串拼接构建 SQL 语句
  - AI 生成的 SQL 仅允许 SELECT 语句，禁止 DML/DDL 操作

6.4 敏感数据过滤

AI 查询接口内置敏感数据过滤机制：

  过滤关键词：密码、手机号、身份证、银行卡、隐私等
  匹配到敏感关键词时返回脱敏提示
  不执行涉及隐私数据的 SQL 查询

6.5 CORS 跨域控制

系统配置 CORS 中间件，控制跨域请求：

  开发模式：允许所有来源（Access-Control-Allow-Origin: *）
  生产模式：可配置允许的来源列表
  允许的方法：GET、POST、PUT、DELETE、OPTIONS
  允许的头：Content-Type、Authorization

6.6 统一错误处理

系统实现全局异常捕获中间件：

  - 捕获所有未处理的异常
  - 生产环境隐藏堆栈信息
  - 返回统一格式的错误响应
  - 记录完整的错误日志供排查


7. 性能优化说明

7.1 异步全链路

从路由层到数据库层全部采用 async/await 异步编程模型：
  - FastAPI 原生异步路由处理
  - SQLAlchemy 2.0 异步 ORM
  - aiomysql 异步 MySQL 驱动
  - asyncio 事件循环管理

7.2 数据库连接池

  - 核心连接数：10
  - 最大溢出连接数：20
  - 连接回收时间：3600 秒
  - 连接预检测：启用

7.3 响应缓存

  - 缓存键基于函数名和参数自动生成
  - 缓存过期自动清除

7.4 分批导出

数据导出采用分批查询策略：
  - 每批查询 5000 条记录
  - 使用 offset/limit 分页

7.5 查询优化

用户行为分析接口的查询优化：

9. 常见问题与故障排除

9.1 启动问题

问题：启动时提示"ModuleNotFoundError"
解决：确认已激活虚拟环境并安装所有依赖
  pip install -r backend/requirements.txt

问题：启动时提示"Connection refused"（数据库连接失败）
解决：检查 MySQL 服务是否启动，.env 中数据库密码是否正确
  mysql -u root -p -e "SELECT 1"

问题：端口 8000 被占用
解决：更换端口或关闭占用进程
  python -m uvicorn backend.main:app --port 8001

9.2 认证问题

问题：AI 查询接口返回 401
解决：确认请求头中包含有效的 Authorization: Bearer <token>
  先调用 /api/auth/login 获取 Token

问题：Token 过期
解决：重新登录获取新 Token，或调用 /api/auth/refresh 刷新

9.3 AI 查询问题

问题：AI 查询超时
解决：检查 LLM_API_KEY 是否有效，网络是否可访问 DashScope API
  超时时间已设置为 120 秒

问题：AI 查询返回"敏感信息"提示
解决：修改查询措辞，避免使用密码、手机号等敏感关键词

9.4 性能问题

问题：导出大数据量时内存不足
解决：系统已实现分批查询（5000条/批），如仍有问题可减小批次大小

问题：查询响应慢
解决：确认已执行 optimize_indexes.sql 创建数据库索引
  mysql -u root -p < backend/sql/optimize_indexes.sql

9.5 数据问题

问题：数据表为空
解决：确认已执行数据导入脚本
  source sql/import_data.sql

问题：数据与 README 中指标不一致
解决：确认使用的是清洗后的数据（100,286 条），而非原始数据（102,318 条）
